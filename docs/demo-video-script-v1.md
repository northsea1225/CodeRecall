# CodeRecall 演示视频分镜脚本 (V1)

**总时长**: 约 5 分钟 (300 秒)

## 演示数据准备要求
为了保证演示的连贯性和视觉丰富度，需要通过 `seed_demo_data.py` 预先准备以下数据：
1. **错题库**: 包含至少 15-20 条错题，涵盖 Python, JavaScript, TypeScript, SQL 等多种语言。
2. **复习状态**:
   - 必须包含至少 1-2 条**今日到期 (Due Today)** 的错题。
   - 至少有 1 条错误代码与正确代码差异明显（适合展示 DiffViewer）的错题，建议使用一段有典型边界条件遗漏的 Python 算法题或 React Hooks 错误。
3. **统计数据**: 过去 30 天的复习记录，确保热力图 (Heatmap) 上有丰富的颜色渐变，且包含过去 7 天的折线图趋势，以及明确的薄弱标签（如 `react`, `algorithm`, `async`）。

## 拍摄与录制注意事项
- 确保浏览器全屏，隐藏书签栏和浏览器扩展图标。
- 录屏软件设置分辨率为 1080p (1920x1080) 或 4K，帧率 60fps 以保证滚动和流式动画平滑。
- **避免**在加载过程中快速点击其他按钮，预留给系统自然过渡的时间。
- **避免**网络波动，正式录制前请先测试一遍 AI 接口 (wzw.pp.ua) 的连通性和响应速度。

## 分镜脚本详细时间表

### 1. 开场与 Dashboard (0:00 - 0:30)
- **时间**: 0:00 - 0:30 (30秒)
- **画面描述**: 启动页面，停留在光亮模式下的 Dashboard 首屏。顶部显示用户信息和概览数据。中间显眼位置有一个 "今日待复习" 的大卡片，显示 `N 题到期`。
- **鼠标操作**: 鼠标在 Dashboard 上缓慢移动，划过今日复习卡片和最近添加的错题列表，稍作停留。
- **旁白台词**: “大家好，欢迎演示 CodeRecall 编程错题本。开发者每天都在踩坑，而 CodeRecall 帮助我们将这些经验沉淀为长期记忆。现在我们在 Dashboard 页面，可以看到系统提醒我今天有 N 道错题需要复习。”

### 2. 进入复习队列 (0:30 - 1:00)
- **时间**: 0:30 - 1:00 (30秒)
- **画面描述**: 点击卡片上的 "开始复习" 按钮（使用 `due_first` 排序）。页面平滑过渡到复习界面。此时仅展示错题的题目描述、Tag 和当时写错的代码片段 (Monaco Editor)。
- **鼠标操作**: 鼠标点击 "开始复习"。进入复习页后，鼠标光标高亮选定错题代码中的几行，模拟思考过程。
- **旁白台词**: “点击开始复习，系统采用了 SM-2 记忆算法，优先为我推送今天最需要回顾的题目。首先，我只能看到题目和我当时写错的代码。这逼迫我先进行主动回忆，思考当时到底错在哪里。”

### 3. 翻牌揭答与 Diff 对比 (1:00 - 1:45)
- **时间**: 1:00 - 1:45 (45秒)
- **画面描述**: 点击 "查看答案" 按钮。下方展开 DiffViewer 区域，左侧为原始错误代码（红色高亮减号），右侧为正确代码（绿色高亮加号）。
- **鼠标操作**: 点击 "查看答案" 后，鼠标在 DiffViewer 的差异高亮处上下滚动，停留展示代码的不同之处。
- **旁白台词**: “经过思考，我们点击查看答案。CodeRecall 提供了直观的 Diff 视图，左边是我当初的错误版本，右边是标准答案。通过对比，我可以清晰地看到自己在边界条件处理上的遗漏。”

### 4. 触发 AI 深度分析 (1:45 - 2:45)
- **时间**: 1:45 - 2:45 (60秒)
- **画面描述**: 点击 "AI 深度分析" 按钮。侧边栏（或下方展开区域）出现 AI 分析面板。文字开始以流式 (Streaming) 效果打字机般逐字输出。输出内容严格分为 4 个小节：根因判断、正确思路、被忽略的边界、复习检查清单。
- **鼠标操作**: 点击 "AI 深度分析"。在 AI 输出过程中，鼠标不要乱动，随文字输出的节奏缓慢滚动面板。
- **旁白台词**: “如果仅仅看答案还不够，我们可以求助 AI。点击 AI 分析，系统会结合题目、我的错解和正解进行针对性诊断。这里使用了流式输出，AI 会给出精准的根因判断，梳理正确思路，点出我忽略的边界，并生成专属的复习检查清单。这就像有一位资深导师在进行代码 Review。”

### 5. 评分反馈与 Toast 提示 (2:45 - 3:15)
- **时间**: 2:45 - 3:15 (30秒)
- **画面描述**: AI 分析阅读完毕，鼠标移动到页面底部的四个评分按钮 (Again, Hard, Good, Easy)。点击 "Good"。触发系统提示，屏幕右上角或中下方弹出 Toast 提示：“下次复习：N天后”。
- **鼠标操作**: 悬停在 "Good" 按钮上展示 Tooltip，然后点击。
- **旁白台词**: “复习完成后，我需要对这次回忆的清晰度进行评分。我觉得这次理解得很透彻，给出一个 Good。SM-2 算法会自动更新这道题的下次复习时间，Toast 提示告诉我已经安排在几天之后了。”

### 6. 切换 Dark Mode 暗黑模式 (3:15 - 3:45)
- **时间**: 3:15 - 3:45 (30秒)
- **画面描述**: 此时完成了今日的所有复习（假设只有一题到期，或者选择回到首页）。鼠标移动到顶部导航栏，点击 Dark Mode 切换开关（月亮图标）。整个页面的背景、Monaco 编辑器、Diff 视图平滑过渡到深色主题。
- **鼠标操作**: 移动到右上角，点击切换主题。在 Dark Mode 下稍微滚动一下 Dashboard 页面展示整体效果。
- **旁白台词**: “作为一款面向开发者的工具，暗黑模式必不可少。我们在顶栏一键切换，可以看到整个界面，包括代码编辑器和对比视图，都实现了完美的暗黑主题适配，保护开发者的视力。”

### 7. 数据统计 Stats 展示 (3:45 - 4:30)
- **时间**: 3:45 - 4:30 (45秒)
- **画面描述**: 点击顶部导航栏进入 "Stats" (统计) 页面。页面加载并展示丰富的图表：顶部是类似 GitHub 的年度复习热力图，下方左侧是近期复习趋势折线图，右侧是薄弱知识点标签排行列表。
- **鼠标操作**: 鼠标移动到热力图的某几个色块上，展示悬浮的 Tooltip（某日复习 N 题）。然后指引向趋势图和薄弱点列表。
- **旁白台词**: “最后，我们进入数据统计中心。这里有我们复习活动的 GitHub 风格热力图，记录着每一天的坚持。趋势折线图展示了复习状态的走向，而薄弱点雷达则精准定位了我常错的技术栈，比如这里显示我在异步处理和 React Hooks 上经常犯错，这为未来的专项练习指明了方向。”

### 8. 演示结束与收尾 (4:30 - 5:00)
- **时间**: 4:30 - 5:00 (30秒)
- **画面描述**: 点击导航栏返回 "Dashboard" 首页。此时 "今日待复习" 卡片上的数字变为 0，并显示一个绿色的完成状态（如 "今日已完成 / All Caught Up!"）。
- **鼠标操作**: 回到首页，鼠标在 "今日已完成" 卡片周围画个圈示意。
- **旁白台词**: “回到首页，今日的复习任务已经清零。通过 CodeRecall，我们让错误不再是简单的挫败，而是技术成长的阶梯。感谢大家的观看，欢迎体验 CodeRecall。”
- **画面结束**: 渐隐至 CodeRecall Logo。
