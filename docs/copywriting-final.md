# CodeRecall 全局文案规范与收口 (W2 Final)

## 一. 总览
本文档合并了先前 `docs/copywriting.md` 和 `docs/copywriting-v2.md` 的所有内容，作为 W2 结项前的最终文案规范参考。未来的所有 UI 实现均以本文档为准，若有冲突，优先遵循本文档的标准。之前的两份文档将作为历史记录留作归档。

## 二. 全局规则（重申）
- **语言基调**：以中文为主，对于常见的英文技术术语（如 Review, Bug, API Key 等）予以保留。
- **错误提示**：所有的错误文案必须包含 CTA（Call To Action，明确的行动建议或指引）。
- **成功反馈**：成功态的 toast 提示必须以动词开头，且总长度控制在 10 个字以内（如"保存成功"、"删除完成"）。
- **空态设计**：空状态必须具有引导性，不可仅显示"暂无数据"。
- **术语规范**："API Key" 首字母必须大写且中间包含空格；环境变量如 `.env` 必须使用反引号包裹。
- **数字格式**：所有文本中的数值均使用阿拉伯数字表示。

## 三. 完整场景文案树
分为以下六个大类进行统一收口：
- **搜索与筛选**：
  - placeholder："搜索题目、标签或内容..."
  - 清除按钮："清除"
  - 分类/语言默认选项："全部"
  - 搜索无结果："未找到匹配的题目，换个关键词试试？"
  - 筛选无结果："当前分类下暂无题目。"
  - 加载中："加载中..."
- **Review 流程**：
  - 空态无到期题："太棒了！今天没有需要复习的题目。去 [添加新题目] 吧！"
  - 开始欢迎："准备好开始今天的复习了吗？共有 N 题待复习。"
  - 题面提示："请在脑海中构思或在草稿纸上写下你的解答。"
  - Show Answer 按钮："查看答案"
  - Code Diff 头："你的回答与标准答案的对比："
  - Self-Rate 4 按钮及 tooltip：
    - Again（忘记了，很快再复习）
    - Hard（想起来很吃力）
    - Good（顺利想起来了）
    - Easy（太简单了，很久以后再复习）
  - 完成页三种情况：
    - 满分："完美！今天的复习任务已全部清空。"
    - 有 Again："还有 N 道题需要加强，继续加油！"
    - 第一题鼓励："良好的开端是成功的一半！"
  - 提交失败："复习记录提交失败，请重试。"
- **Review 键盘快捷键 hint 卡片**：
  - `Space`: 显示答案
  - `1-4`: 评分 (Again-Easy)
  - `Esc`: 退出复习
  - `R`: 查看原题
  - `H`: 唤起帮助
- **MistakeEditor**：
  - 新建和编辑按钮文案："新建题目"、"编辑题目"
  - 字段 placeholder："请输入题目描述"、"粘贴你的代码..."
  - 必填校验："此项为必填项"
  - 保存 toast："题目保存成功"
- **全局 Toast**：
  - 保存确认："内容已保存"
  - 删除确认："确认删除此项吗？"
  - 导入三态："导入准备中..."、"导入成功 N 条记录"、"导入失败，请检查文件格式"
  - 导出成功："导出文件已生成"
  - 网络断开 banner："网络连接已断开，请检查您的网络设置。"
- **AI 分析**：
  - 按钮四态："AI 分析"、"分析中..."、"重新分析"、"分析完成"
  - Tooltip："让 AI 帮你找出代码中的潜在问题"
  - 成功 toast："AI 分析完成"
  - 异常文案四种：
    - 网络："网络请求超时，请稍后重试。"
    - 鉴权："API Key 无效，请检查配置。"
    - 额度："API 额度已耗尽，请充值或更换 Key。"
    - 限流："请求过于频繁，请稍候再试。"
  - 配置引导 Banner："尚未配置 AI 助手？[去设置] API Key 体验智能分析。"

## 五. 给 Codex 的实施清单
- **文案常量化**：所有文案常量统一存放在 `frontend/src/i18n/zh.ts` 文件中（W2 阶段先全面支持中文，W4 阶段再考虑英文国际化）。
- **杜绝硬编码**：组件内部绝对不允许硬编码中文字符串，必须统一从 i18n 对象中读取。
- **动态数据注入**：带有动态数字（如 N 题、M 次）的文案，使用模板字符串进行变量注入。
- **情感化随机**：情感化鼓励语可以配置为数组，运行时随机从 3 个候选文案中选取一个，避免每次看到相同的话语产生单调感。
- **Toast 交互**：错误类的 toast 提示统一使用 antd 的 notification API，并且展示时间至少保留 5 秒，确保用户有足够时间阅读。

## 六. 验收
- W2 所有页面的界面文案均能通过 `frontend/src/i18n/zh.ts` 找到对应的 key。
- 代码库中不存在遗留的硬编码中文字符串（强烈建议引入 eslint 中文硬编码检查规则作为保障）。
