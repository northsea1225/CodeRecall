# CodeRecall Demo Script V2 — 5 分钟演示视频脚本

**版本**: V2（基于实际 UI 代码审查，2026-04-20）
**总时长**: 约 5 分钟
**语言**: 普通话口语旁白
**路径**: 启动 → Dashboard → 开始复习 → 题目卡片 → 显示答案 → DiffViewer → AI 流式分析 → 自评打分 → 会话总结 → Stats 页图表

---

## 演示前准备

- 确保数据库中有 **15-20 条错题**，涵盖 Python、JavaScript、TypeScript
- 今日到期题目至少 **3 条**（确保 Dashboard 按钮显示"今日有 N 题到期"）
- Stats 页面有至少 **30 天**复习历史，确保折线图有明显趋势
- 浏览器宽度 1440px，左侧边栏已展开，字体缩放 100%
- 隐藏书签栏，关闭无关标签页

## 场景二：从 Dashboard 发起复习

| 字段 | 内容 |
|------|------|
| **时间** | 0:40 – 1:10 |
| **操作** | 鼠标移到顶部右侧按钮，按钮显示"今日有 N 题到期"（ghost primary 按钮），点击它；页面跳转到 `/review` |
| **画面** | 按钮文案根据实际到期数量动态生成；点击后 review store 调用 `startSession({ strategy: "due_first" })`，跳转 Review 页并开始加载第一题 |
| **旁白** | "注意到了吗，这个按钮直接告诉我今天有几道题该复习了。这背后是 SM-2 间隔重复算法在运作——它会根据你上次的评分，自动推算出每道题最合适的复习时间。我点一下，直接进入今天的复习队列。" |
| **亮点镜头** | 是 — 放慢鼠标悬停在按钮上，让观众看到按钮文案细节 |

## 场景四：翻牌 — DiffViewer 代码对比（亮点）

| 字段 | 内容 |
|------|------|
| **时间** | 1:45 – 2:30 |
| **操作** | 点击「显示答案」按钮（或按空格键）；页面切换到 AnswerView，DiffViewer 呈现左右分屏 |
| **画面** | Monaco DiffEditor：左侧标题「错误实现 (Wrong)」——红色高亮删除行；右侧「正确实现 (Correct)」——绿色高亮新增行；语言标注在右上角；上方还有元数据 Tags 行 |
| **旁白** | "你看，这是 CodeRecall 最核心的功能之一——代码对比视图。左边是我当时写错的代码，右边是正确答案。红色的是我多余或者错误的部分，绿色的是应该有但我漏掉的。这种视觉对比，比直接看文字描述要直接得多，一眼就能定位问题在哪行。" |
| **亮点镜头** | 是 — 放慢速度，鼠标在红色删除行和绿色新增行之间来回移动，停留 3-4 秒 |

## 场景六：SM-2 自评打分

| 字段 | 内容 |
|------|------|
| **时间** | 3:30 – 3:55 |
| **操作** | AI 分析结束后，鼠标移至页面底部的 SelfRateGroup；四个按钮依次为：1. Again（danger 危险色）/ 2. Hard / 3. Good（primary）/ 4. Easy；鼠标悬停在 "3. Good" 按钮上展示 Tooltip，点击 |
| **画面** | 底部提示文字："按 1 / 2 / 3 / 4 也可完成自评"；点击 Good 后按钮进入 loading；提交完成，进度条更新，加载下一题或进入 CompletedView |
| **旁白** | "看完分析，我来给自己打个分。这四个按钮对应的是 SM-2 算法里的掌握程度评级——Again 表示完全没想起来，Hard 是想起来了但很费力，Good 是基本掌握，Easy 就是轻松搞定。我选 Good，系统会把这道题安排在几天后再次出现。" |
| **亮点镜头** | 否 |

## 旁白节奏提示

- 说话语速：约 180-200 字/分钟（正常对话速度，不要太快）
- 每个"亮点镜头"场景，鼠标操作速度比平时慢 30%
- 场景 5（AI 流式输出）期间基本不需要操作，重点跟随文字输出的节奏轻微滚动
- 场景切换时（如点击导航菜单）给页面 1-1.5 秒过渡时间再开始旁白
