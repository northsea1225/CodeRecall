# CodeRecall 界面文案收口规范 V2

本规范针对 W2 阶段新增的 AI 分析、首次体验以及异常状态等场景，补充并修正了 Day 1 的底层文案。

## 一. AI 分析场景文案

### 操作按钮
*   **就绪态**: `⭐ AI 深度分析`
*   **流式态**: `思考中...`
*   **完成态**: `AI 分析完成`
*   **禁用态**: `AI 分析未启用`

### 提示与反馈
*   **禁用 Tooltip**: `请在后端 .env 配置 LLM_API_KEY 后重启服务`
*   **成功 Toast**: `复制成功，已拷贝到剪贴板`
*   **异常拦截文案**:
    *   网络波动: `AI 服务暂时不可用，稍后重试`
    *   鉴权失败: `API Key 无效，请检查后端配置`
    *   额度不足: `API 额度不足，请充值或更换模型`
    *   高频限流: `请求过于频繁，稍后再试`

## 二. 配置引导文案

当首次进入 Review 页且接口检查发现 AI 能力尚未开启 (capability 的 enabled 为 false) 时，在页面顶部提供横幅引导：

*   **Banner 主文案**: `想开启 AI 错因分析？到 backend 根目录的 .env 添加 LLM_API_KEY 并设置 ENABLE_AI_ANALYSIS 为 true 重启后端即可。`
*   **交互按钮 (CTA)**:
    *   主推操作: `查看文档` (跳转至 docs 的 index 页面)
    *   关闭操作: `不再提示` (点击后状态写入客户端 localStorage)

## 三. Review 流程微文案补齐

在复习核心链路中，针对不同进度节点提供情感化鼓励：

*   **初始状态**: 开始复习第 1 题时显示 `开始你的第一题，加油`
*   **完美通关**: 完成 N 题且 Again 次数为 0 时显示 `满分表现，所有题都过了`
*   **存在错题**: 完成 N 题且 Again 次数大于 0 时显示 `N 题复习，M 题需要再来一次，已经不错了` (注：N 和 M 需要在代码中用真实变量替换)

## 四. MistakeEditor 保存动作文案统一

在错题编辑与新建表单中，按钮文案及轻提示需严格统一：

*   **表单按钮**:
    *   新建模式: `保存错题`
    *   编辑模式: `保存修改`
*   **执行状态**:
    *   保存中: `正在保存`
    *   成功 Toast: `已保存`
    *   失败反馈: `保存失败，请检查网络后重试`

## 五. 文案全局补充规范

1.  所有提及大模型密钥的术语，统一拼写为首字母大写的 `API Key`（中间包含一个半角空格）。
2.  界面或提示词中凡是涉及到环境变量文件的，一律使用反引号包裹写为 `.env`。
3.  统计题数、次数等状态变量时，一律使用阿拉伯数字，不可使用中文数字（例如写"3 题"而不是"三题"）。
