## 2. W4 入口指引（按优先级排序）

W4 是项目的“冲刺与发布”周，重点在于演示准备与收尾。请严格按照以下优先级展开工作：

1. **准备高质量演示数据 (`seed_demo_data.py`)**
   - 现有的 mock 数据可能不够生动。W4 第一要务是更新或执行 seed 脚本。
   - 要求入库：至少 30 天的逼真复习历史（为了让 Stats 热力图好看）、20+ 条高质量错题（涵盖 Python, TS, SQL 等不同语言和典型错因）。
2. **确认 AI API Endpoint 可用性**
   - **关键风险点**：演示高度依赖网络环境和外部 API (`wzw.pp.ua`)。
   - 请在 W4 初期和演示前多次测试该接口的连通性、延迟以及流式输出的稳定性。
3. **结合分镜脚本彩排与 UX 微调**
   - 参照 `docs/demo-video-script-v1.md` 进行完整流程的彩排。
   - 在彩排中捕捉肉眼可见的 UX 瑕疵（如：按钮偏移、动画卡顿、字体间距不一致、特定数据下的溢出等），并进行修复。
4. **录制 5 分钟演示视频**
   - 保证网络良好、无干扰的情况下，按照脚本录制最终视频（确保 1080p+ 清晰度）。
5. **准备最终答辩材料**
   - 整理项目架构图、核心难点总结（如 SM-2 算法实现、流式 AI 对接、Monaco 优化）以及 Readme 润色。

## 3. 已知遗留问题与技术债

在交接前，团队需知晓以下非阻塞但需要关注的问题：

- **AI Endpoint 网络依赖**: 如前所述，当前未提供完全的本地 fallback 机制，一旦 API 超时，AI 分析面板可能白屏或报错。需要在演示时确保网络，或在 W4 酌情增加友好的超时错误提示。
- **Prompt v2 的真实验证**: W3 更新了更严格的 Prompt 规范，但在多语言和极其复杂的错题上，AI 偶尔是否会跑偏（不严格遵循 4 段式）还需要在 W4 通过真实多语言 demo 数据做最后验证。
- **前端依赖安全警告**: 运行 `npm install` 时存在中等 (moderate) 级别的漏洞警告。评估为非阻塞性问题，不影响核心功能与演示，如 W4 时间充裕可酌情升级依赖解决。
