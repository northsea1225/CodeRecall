# CodeRecall 文案库

## 1. 搜索与筛选（MistakeList）
- **搜索框 placeholder**：搜错题题干、错因、代码片段...
- **清除筛选按钮**：重置筛选
- **分类筛选默认选项**：全部分类
- **语言筛选默认选项**：全部语言
- **搜索无结果空态**：
  - Title：未找到匹配的错题
  - Description：尝试更换搜索词，或清除过滤条件。
  - CTA：重置搜索
- **筛选无结果空态**：
  - Title：当前筛选条件下无错题
  - Description：尝试放宽语言或分类筛选条件。
  - CTA：清除筛选
- **筛选 pending 态**：正在检索错题...

## 2. Review 页（W2 核心）
- **无到期题空态**：
  - Title：恭喜！今天没有需要复习的错题
  - Description：休息一下，或去录入新的错题。
  - CTA：去录入
- **会话开始欢迎语**：开始今日复习 (进度 0/N)
- **题目展示提示**：请回想正确解法与错因
- **Show Answer 按钮**：显示答案
- **Code Diff 头部文案**：左侧展示「错误实现 (Wrong)」，右侧展示「正确实现 (Correct)」
- **Self-Rate 按钮与 Tooltip**：
  - 重来 (Again) —— 提示: `< 1m`
  - 困难 (Hard) —— 提示: `12h`
  - 良好 (Good) —— 提示: `1d`
  - 简单 (Easy) —— 提示: `4d`
- **Session 完成页**：
  - Title：复习完成！
  - Description：本次复习 N 题，正确率 M%。继续保持！
  - CTA：返回控制台
- **提交失败重试态**：提交评价失败，点击重试

## 3. 全局 Toast / 错误 / 成功反馈
- **错题保存**：保存成功 / 保存失败，请检查网络连接
- **删除确认弹窗**：
  - Title：确认删除此错题？
  - Body：删除后不可恢复，相关的复习记录也将一并清除。
  - CTA：取消 / 确认删除
- **导入/导出**：
  - 导入成功：成功导入 N 条错题
  - 部分跳过：导入完成，N 条成功，M 条已跳过
  - 导入失败：导入失败，请检查文件格式
  - 导出成功：导出成功，已保存至本地
- **网络 banner**：当前处于离线状态，部分功能可能不可用

## 4. MistakeEditor 补充文案
- **字段 Placeholder**：
  - Title：一句话总结这个错误（如：React useEffect 依赖项遗漏）
  - Stem：描述问题发生的场景或具体需求...
  - Wrong：贴上错误的代码实现...
  - Correct：贴上正确的代码实现...
  - Reason：分析为什么会错，以后如何避免...
- **必填校验错误**：标题不能为空；错误与正确代码至少填写一项
- **保存 Toast**：正在保存... / 错题已保存

## 5. 文案原则
- 中文为主，英文术语不翻译（如 "Markdown"、"JSON"、"Monaco Editor"）
- 错误文案：说明原因 + 给 CTA，避免单薄表达（如"操作失败"）
- 成功 toast：动词开头，≤ 10 字
- 空态：引导式文案，不要只说"暂无数据"
