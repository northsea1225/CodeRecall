# CodeRecall 录屏指南 (macOS)

**适用场景**: 录制 5 分钟演示视频，用于项目汇报或对外展示
**平台**: macOS (Ventura 13+ / Sonoma 14+ / Sequoia 15+)

---

## 1. 推荐工具：QuickTime vs OBS

### QuickTime Player（系统内置）

**优点**
- 零安装，macOS 自带，直接用
- 录屏操作极简：菜单 → 文件 → 新建屏幕录制，三步完成
- 音频配置简单（选麦克风即可），无需额外调音
- 导出格式为 .mov，后续用 iMovie 或 CapCut 直接打开

**缺点**
- 不支持直接输出 MP4，需导出后转码
- 无法添加叠加层（如摄像头画中画、计时器）
- 不支持实时调整码率
- 录制区域只能全屏或框选，无法精确到像素

**适合情况**: 第一次录制、追求简洁流程、不需要摄像头画面

---

### OBS Studio（免费开源）

**优点**
- 支持直接录制为 MP4/MKV，省去转码步骤
- 可配置视频质量（CRF 值）、码率上限
- 支持添加多个 Source 叠加（浏览器窗口 + 摄像头 + 文字水印）
- 录制区域可精确指定窗口或自定义分辨率

**缺点**
- 第一次配置需要 10-15 分钟
- 对初学者有学习曲线（场景/来源/混音的概念）
- macOS 下需授予屏幕录制和麦克风权限

**适合情况**: 需要更精细控制质量、想添加摄像头或水印

---

### 本项目推荐

**首选 QuickTime**，适合非录制专业人员。完整步骤：

1. 打开 QuickTime Player
2. 菜单栏 → 文件 → 新建屏幕录制（快捷键 `Control + Command + N`）
3. 点击下拉箭头，选择麦克风来源
4. 点击红色录制按钮 → 选择「录制所选屏幕部分」→ 框选浏览器窗口
5. 录制完毕后点击菜单栏录制图标停止

**如果你对视频质量有较高要求（如 4K 提交）**，改用 OBS。

## 2. 画面参数

### 分辨率

| 选项 | 推荐度 | 说明 |
|------|--------|------|
| **1440p (2560×1440)** | 首选 | 适合 MacBook Pro 14/16 Retina 屏，细节清晰，文件不过大 |
| **1080p (1920×1080)** | 备选 | 普通显示器或需要控制文件大小时使用 |
| 4K (3840×2160) | 不推荐 | 文件过大，分享和剪辑均变慢，除非有 4K 提交要求 |

macOS 上实际录制分辨率跟随屏幕物理像素，若是 Retina 屏幕，QuickTime 默认会以 2× 分辨率录制，注意在导出时选择合适的尺寸。

### 帧率

- **推荐 60fps**：CodeRecall 的 AI 流式输出（逐字打字效果）和进度条动画在 60fps 下更流畅
- 如果剪辑软件或上传平台限制了 30fps，则降到 30fps，差异可接受

### 浏览器窗口大小

```
推荐宽度: 1440px
推荐高度: 900px（或最大化窗口）
浏览器: Chrome 或 Safari（推荐 Chrome，Monaco Editor 兼容性更好）
```

操作方法（Chrome）：
1. 打开开发者工具 → 点击切换设备模式图标（Responsive）
2. 手动输入 `1440 × 900`
3. 关闭开发者工具，此时窗口已固定为该尺寸

或使用 macOS 窗口管理：System Settings → Desktop & Dock → 关闭窗口自动排列，手动拖拽到合适大小。

### 缩放级别

- 浏览器缩放保持 **100%**（`Command + 0` 重置）
- 如果在外接显示器上录制，确认系统缩放是 100% 而非 125% 或 150%
- Monaco Editor 字体大小在代码中已固定，不需要额外调整

## 3. 鼠标高亮

演示视频中鼠标光标需要清晰可见，以下是 macOS 可用的免费工具：

### KeyCastr（推荐，免费开源）

- GitHub: `github.com/keycastr/keycastr`
- 功能：实时在屏幕左下角显示当前按键输入（如 `Space`、`1`、`2` 等）
- 特别适合 CodeRecall 演示：SM-2 评分时按键盘 `1/2/3/4` 评分，观众能看到按了哪个键
- 安装：`brew install --cask keycastr` 或从 GitHub Releases 下载

### Cursor Pro（付费，$10/年）

- 提供鼠标点击动画（高亮圆圈）和鼠标轨迹高亮
- 如果预算允许，视觉效果最佳

### macOS 内置无障碍缩放（免费）

- System Settings → Accessibility → Zoom → 开启「Use scroll gesture with modifier keys to zoom」
- 演示时按住 Control 滚轮即可局部放大，适合在 DiffViewer 处放大代码细节
- 缺点：鼠标本身不高亮，只是放大画面

### 综合建议

仅需免费方案时：**安装 KeyCastr 显示按键** + **系统光标尺寸调大**（System Settings → Accessibility → Display → Pointer Size → 拖到中等偏大）即可满足需求。

## 4. 剪辑工具

### iMovie（系统内置，免费）

**推荐给本项目的非专业剪辑用户**

- 无需学习曲线，拖进视频文件 → 剪掉开头结尾停顿 → 加字幕 → 导出
- 支持直接导出 1080p/4K MP4
- 支持添加文字标注（可标注"亮点镜头"字幕卡）
- 不支持复杂特效，但 5 分钟演示视频完全够用

**使用步骤**：
1. 打开 iMovie → 新建项目 → 导入录屏视频
2. 拖到时间轴 → 剪掉前后准备段落
3. 需要标注的地方：添加 Title（标题字幕），选择下方滚动条样式
4. 文件 → 共享 → 文件，分辨率选 1080p，质量选「高」

---

### CapCut（免费，需账号）

**适合如果需要添加中文字幕或 BGM**

- 自动语音识别字幕功能很强，可以一键生成旁白字幕
- 有丰富模板和转场效果
- 手机端和桌面端都可用
- 缺点：需要注册账号，部分高级功能需要付费，界面有广告推荐

---

### DaVinci Resolve（免费版功能强大）

**适合对视频质量有追求的用户，但学习成本高**

- 专业级调色和剪辑工具
- 免费版功能已非常完整
- 学习曲线较陡，对于一个 5 分钟演示视频来说有些大材小用
- 推荐：如果你以后经常需要剪视频，投资学习 DaVinci 是值得的；否则用 iMovie

### 推荐顺序

```
一次性演示视频 → iMovie（最快）
需要字幕 → CapCut
长期做内容创作 → DaVinci Resolve
```

## 5. 导出参数

### 格式

- **MP4 (H.264)**：兼容性最好，适合提交、上传各类平台
- 避免使用 .mov 直接提交（文件较大，部分平台不支持）
- 如果使用 QuickTime 录制得到 .mov，用 iMovie 或 HandBrake（免费）转换为 MP4

### 推荐导出设置

| 参数 | 推荐值 |
|------|--------|
| 格式 | MP4 / H.264 |
| 分辨率 | 1920×1080（1080p）|
| 帧率 | 30fps（已足够）|
| 视频码率 | 8 Mbps（1080p 高质量）|
| 音频格式 | AAC |
| 音频码率 | 128 kbps |

### 文件大小估算

- 5 分钟 × 8 Mbps ≈ **300 MB**（1080p 高质量）
- 5 分钟 × 4 Mbps ≈ **150 MB**（1080p 普通质量，多数平台可接受）

### 控制文件大小

1. iMovie 导出时选「中等」质量 → 约 150-200 MB
2. 或使用 HandBrake 转码，将 CRF 设为 22-24（值越大文件越小，质量略降）
3. 如平台有 100MB 限制：CRF 设 26，码率限制 3 Mbps，5 分钟约 110MB

## 6. 彩排建议：三次彩排协议

正式录制前，按以下三轮彩排流程执行。**每次彩排都要完整走一遍 demo 路径**，不要中途停下来修改。

---

### 第一次彩排：流程熟悉

**目标**: 确认每一步操作都能顺利完成，不出现意外错误

**检查项**:
- [ ] 后端服务已启动，API 正常响应
- [ ] LLM_API_KEY 已配置，AI 分析按钮可以正常触发
- [ ] 今日到期题目数量 ≥ 3 条（否则 Dashboard 按钮文案异常）
- [ ] Stats 页面有足够历史数据（折线图/热力图有数据，不显示"暂无数据"）
- [ ] 走完完整路径：Dashboard → 点击到期按钮 → StemView → 显示答案 → DiffViewer → AI 分析 → 自评打分 → CompletedView → Stats

**不需要关注**: 时间是否准确、鼠标是否平滑

---

### 第二次彩排：时间控制

**目标**: 确保每个场景的时间在规定范围内，总时长在 4:45 - 5:15 之间

**操作**:
- 打开手机秒表
- 按照 docs/demo-video-script-final.md 的时间轴，卡表演练
- 在每个场景切换点报一下当前时间

**常见问题处理**:
- AI 分析输出时间不稳定（依赖网络）：提前测试 AI 响应速度，AI 输出期间可以补充旁白，不要冷场
- CompletedView 出现时间不可控：如果错题很多，第二次彩排可以只复习 2-3 道题后直接跳到 Stats 页

---

### 第三次彩排：鼠标流畅度

**目标**: 确保鼠标移动平稳、不抖动、不乱点

**练习要点**:
- 鼠标从一个元素移到另一个元素时，走直线，速度匀速
- 在"亮点镜头"标注的场景，主动减慢鼠标速度（大约 0.5x 平时速度）
- 点击操作前，先将鼠标停在目标元素上 0.5 秒再点击（避免观众没看清就点了）
- 滚动时使用触控板双指匀速滑动，避免惯性滑动过头

**强烈建议**: 第三次彩排录制下来（不用保留），回放检查鼠标是否有抖动，特别关注 DiffViewer 和 Stats 热力图这两个"亮点镜头"场景。
