# 演示 PPT 大纲 (技术受众)

**目标听众**: 懂软件工程的技术同行、导师、评审专家。

## Slide 1: 标题页
- **标题**: CodeRecall 码迹 —— 开发者专属的智能错题本与知识沉淀系统
- **副标题**: 借助间隔重复与大语言模型，将 Bug 转化为你的核心竞争力
- **建议视觉**: 应用 Logo，配合一张深色模式下极具极客感的 DiffViewer 界面截图。

## Slide 2: 问题陈述 (为什么程序员需要错题本？)
- 开发者每天都在解决 Bug，但经常在几个月后再次踩到同样的坑。
- 传统的笔记工具（如 Notion/Evernote）对代码排版和差异对比（Diff）支持薄弱。
- 缺乏科学的复习机制，记录下来的经验很快就会遗忘。
- **建议视觉**: 程序员日常抓狂的表情包，对比“凌乱的文本笔记”与“结构化的代码错题库”。

## Slide 3: 整体系统架构
- 采用现代化的前后端分离架构。
- **前端**: React 18 + TypeScript + Vite，提供极速响应的 SPA 体验。
- **后端**: FastAPI + Python 3，提供高性能异步 API 与流式数据处理。
- **数据层**: SQLite + SQLAlchemy ORM，轻量且满足单用户/本地优先部署需求。
- **建议视觉**: 一张精简但清晰的架构拓扑图 (Frontend -> REST/SSE -> Backend -> SQLite/LLM)。

## Slide 4: 核心技术栈细节
- **UI & 编辑器**: 深度集成 Monaco Editor (VS Code 同款内核) 实现精准的代码高亮与 Diff 渲染。
- **数据可视化**: 使用 Recharts 绘制性能优异的统计图表与 GitHub 风格热力图。
- **后端迁移**: Alembic 实现数据库表结构的平滑演进与版本控制。
- **建议视觉**: Tech Stack 的各种技术 Logo 矩阵组合。

## Slide 5: 核心特性 - SM-2 间隔重复算法
- **原理解析**: 基于认知心理学遗忘曲线的 SuperMemo-2 (SM-2) 算法。
- 每次复习根据用户的自我评分 (1-5分)，动态调整下一次出现的间隔 (Interval)、难度系数 (E-Factor) 和重复次数。
- 确保用户只在快要遗忘的边缘复习，最大化学习效率，避免无效刷题。
- **建议视觉**: 艾宾浩斯遗忘曲线示意图，配合 SM-2 的核心状态流转图。

## Slide 6: 核心特性 - AI 深度分析 (流式响应)
- 结合大语言模型 (Claude/Gemini 等)，对错题进行多维度解析（原因、修复、最佳实践）。
- **技术亮点**: 采用 SSE (Server-Sent Events) 实现流式输出 (Streaming)，拒绝漫长等待。
- 精心设计的 Prompt 模板，确保 AI 输出特定于编程领域的专业指导。
- **建议视觉**: 前端自动滚动的流式输出动图 (GIF)，或是 SSE 请求的网络时序图。

## Slide 7: 核心特性 - 专业的代码展示体验
- 专门为代码阅读优化的视图。
- 左右对照或内联样式的 DiffViewer。
- 语法高亮支持主流语言，自适应明暗主题 (Dark/Light mode)。
- **建议视觉**: 占据半个屏幕的一张高质量代码 Diff 对比截图。

## Slide 8: 数据模型与 API 设计
- **核心实体**: `Mistake` (错题主表), `ReviewLog` (复习记录), `Tag` (分类标签)。
- 严格遵循 RESTful 规范的 API 契约设计。
- Pydantic 模型保证了接口输入输出的强类型校验。
- **建议视觉**: 核心数据库表的 ER 图 (Entity-Relationship Diagram)。

## Slide 10: 测试覆盖与工程质量

- **建议视觉**: 测试终端通过 (绿色的 PASSED) 的运行截图或覆盖率报告。
