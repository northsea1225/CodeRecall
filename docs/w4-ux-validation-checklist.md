# W4 UX 验证与体验测试清单 (UX Validation Checklist)

本测试清单旨在进行全面的人工体验走查，确保应用在极端情况和核心交互链路下的稳定性和视觉表现。

## 1. 深色模式 (Dark Mode)
- [ ] **测试说明**: 切换系统或应用至深色模式。
- [ ] **测试方法**: 浏览所有主要页面（仪表盘、错题列表、复习页面、统计数据）。
- [ ] **通过标准**: 
  - 所有图表（Recharts）颜色对比度清晰，无不可见文本。
  - Monaco Editor 自动切换为深色主题且代码高亮正确。
  - 提示框（Tooltips）、弹窗（Modals）背景色与文字颜色适配正确。

## 2. AI 分析流式输出 (AI Streaming)
- [ ] **测试说明**: 针对某条错题触发 AI 分析功能。
- [ ] **测试方法**: 点击“AI 深度分析”按钮，观察内容生成过程。
- [ ] **通过标准**:
  - SSE 流式加载顺畅，无明显卡顿或截断。
  - AI 输出容器在内容不断增加时，能够**自动滚动到底部** (Auto-scroll to bottom)。
  - Markdown 渲染正确，代码块有正确的语法高亮。

## 3. 空状态展示 (Empty State)
- [ ] **测试说明**: 模拟没有错题或所有错题均已掌握（无需复习）的场景。
- [ ] **测试方法**: 清空数据库或将所有错题的下一次复习时间推迟到未来。访问 Dashboard (仪表盘)。
- [ ] **通过标准**:
  - 仪表盘展示友好、鼓励性的空状态提示（如“太棒了！今天没有需要复习的错题”）。
  - 没有出现图表报错或由于数据为 0 导致的布局塌陷。

## 4. 代码差异对比视图 (DiffViewer)
- [ ] **测试说明**: 查看包含超长单行代码的错题 Diff。
- [ ] **测试方法**: 添加或找到一条修改前后代码差异明显，且存在极长单行代码的记录，进入详情页查看。
- [ ] **通过标准**:
  - DiffViewer 必须出现**横向滚动条** (Horizontal scrollbar)，不会撑破外层容器。
  - 删除线（红色）和新增线（绿色）高亮对齐准确。

## 5. 控制台无报错 (Console Errors)
- [ ] **测试说明**: 在完整的使用流中，前端控制台不应有异常抛出。
- [ ] **测试方法**: 打开 Chrome DevTools (F12) 的 Console 面板，完整走一遍：添加错题 -> 列表翻页 -> 复习答题 -> 查看统计。
- [ ] **通过标准**:
  - 全程无红色的 Uncaught Error。
  - 无 React 的 "missing key in list" 警告或内存泄漏 (Memory leak) 警告。

## 6. 复习心流 (Review Flow)
- [ ] **测试说明**: 测试核心的 SM-2 间隔重复复习链路。
- [ ] **测试方法**: Dashboard 查看待复习数量 -> 点击“开始复习” -> 思考后点击“显示答案” -> 提交掌握程度评分 (1-5) -> 自动进入下一题 -> 结束时展示复习总结 (Summary)。
- [ ] **通过标准**:
  - 待复习计数 (due-count) 与实际弹出的题目数量一致。
  - 评分提交后，平滑过渡到下一题，不卡顿。
  - 复习总结页准确显示本次复习的数量和耗时。

## 7. 统计数据实时刷新 (Stats Refresh)
- [ ] **测试说明**: 复习操作完成后，统计图表需实时响应。
- [ ] **测试方法**: 完成一次复习会话后，立即跳转至“统计面板”。
- [ ] **通过标准**:
  - 遗忘曲线图、热力图 (Heatmap) 以及掌握度饼图上的数据已更新。
  - 弱点分析 (Weak areas) 的占比根据最新的评分发生相应变化。
