## 五、核心算法

### 1. 随机抽题（两档）
- **纯随机**（MVP / 演示）：从筛选结果集中随机抽一题。
- **加权随机**（W3+）：
  - 已到期 `+5`
  - 从未复习 `+3`
  - 上次结果为 `again` `+4`
  - 复习次数越多权重越低

### 2. SM-2 简化版间隔重复
评分映射：`again=0, hard=3, good=4, easy=5`

```python
if q < 3:                      # again
    repetition = 0
    interval_days = 1
else:
    if repetition == 0:        interval_days = 1
    elif repetition == 1:      interval_days = 3
    else:                      interval_days = round(interval_days * ease_factor)
    repetition += 1

ease_factor = max(1.3, ease_factor + (0.1 - (5-q) * (0.08 + (5-q)*0.02)))
next_review_at = now + interval_days
```

**落地节奏**：W2 先做「随机抽 + 记 log」；W3 再切「到期优先 + SM-2 调度」，风险最低。

### 3. 关键词搜索（两阶段）
- **MVP**：SQLite `LIKE` 扫 `title / stem / wrong / correct / reason`；关键词按空格拆分 AND，每词任一字段命中即可。
- **优化**：增 `search_text` 冗余列；如环境支持可挂 FTS5（非必须）。

- 技术栈选 **React + FastAPI + SQLite** Web 本地优先方案。

| 议题 | Codex | Gemini | 最终采纳 |
|---|---|---|---|
| 存储方案 | 强调 SQLite 工程可控 | 倾向更轻的本地方案 | SQLite 为主 + JSON 导入导出 |
